We Gave 6 AI Agents a Task. Only 1 Picked the Market Leader.

March 2026 · 5 min read

We ran a simple experiment: give OpenAI Operator, Codex, Claude Code, Devin, Gemini CLI, and GitHub Copilot real tasks and see which services and tools they choose.

"Book me a hotel near downtown Chicago for this weekend."

"Add payment processing to my Node.js app."

"Find me a CRM under $50/month and set up a trial."

We ran these tasks — and 50 variations — hundreds of times across sessions. The results were striking.

Agents don't recommend. They decide.

This is different from asking ChatGPT "what's the best hotel?" An LLM gives you a list. An agent goes and books one. Operator opened a browser, navigated to a booking site, and completed the reservation. The user never saw alternatives.

The market leader in each category — the brand with the biggest marketing budget and highest brand recognition — was the agent's pick in only 17% of tasks.

The takeaway: AI agents don't pick the biggest brand. They pick the service that's easiest for them to interact with — structured data, accessible APIs, machine-readable pricing. Being "well-known" is irrelevant if the agent can't parse your site.

Why does this happen?

1. Agents need structured, machine-readable data. If your pricing is in a PDF or your booking flow requires 7 clicks, the agent routes to a competitor with a clean API.

2. Agent defaults are sticky. Once an agent successfully completes a task with a particular service, it tends to use that service again. First-mover advantage is massive.

3. Developer tools live and die by import patterns. For Codex and Claude Code, the library with the most parseable docs and correct type definitions wins every import statement.

The compounding problem

Every time an agent picks your competitor, that successful interaction reinforces the pattern. The agent learns: "this service works." Your competitor becomes the default. The cycle accelerates, and reversing it gets harder every month.
